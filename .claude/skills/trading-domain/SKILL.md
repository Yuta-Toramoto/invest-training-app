# invest-training-app — ドメインルール

本プロジェクトはデイトレード初学者向けの教育サービス。**金融商品取引法上の投資助言行為に該当しないよう細心の注意**を払う。

## 用語の正式呼称（必ずこれを使う）

| カテゴリ | 用語 | 説明 |
|---|---|---|
| エントリー | 買い (Long) | 価格上昇を期待して買う |
| エントリー | 空売り (Short) | 価格下落を期待して売る |
| エグジット | 利確 / 損切り | 利益確定・損失確定 |
| 価格情報 | ローソク足 / 始値 / 高値 / 安値 / 終値 | OHLC |
| 出来高 | 出来高 (Volume) | 一定期間の取引量 |
| 板情報 | 板 / Order Book | 売買注文の一覧 |
| 指標 | 移動平均線 (MA) / RSI / MACD / VWAP | 主要テクニカル指標 |

**「ドテン」「踏み上げ」「養分」などの俗語**は教育コンテキストでは使わない（正式用語で表記）。

## 出題で扱うチャートパターン

問題作成時はこの一覧の名称で `tags` に登録する:

- ピンバー (pin_bar)
- 包み足 (engulfing)
- ダブルトップ / ダブルボトム (double_top / double_bottom)
- ヘッドアンドショルダー (head_and_shoulders)
- 三角持ち合い (triangle)
- ブレイクアウト (breakout)
- フェイクアウト (fakeout)
- ギャップアップ / ギャップダウン (gap_up / gap_down)
- 移動平均線のゴールデンクロス / デッドクロス (golden_cross / dead_cross)

## 法務上の禁則表現

**絶対に書かない**:
- 「絶対に儲かる」「必ず利益が出る」
- 「推奨銘柄」「おすすめ投資先」
- 「○○円稼げる」（具体的な利益金額の保証）
- 「私の手法をマネすれば」（個別助言と取られる表現）

**必ず添える**:
- 「教育目的のシミュレーションです」
- 「実際の投資判断はご自身の責任で行ってください」
- 「過去のチャートパターンが将来も同じ結果になるとは限りません」

## 問題作成の品質基準

1. **答えに根拠がある**: 「なんとなく」で正解が決まる問題は不可。`explanation` に明確な根拠を書く。
2. **再現性のあるパターン**: 1 つのチャートパターンに紐づく問題は最低 3 問用意し、`tags` で結びつける。
3. **「見送り (no_trade)」を正解にする問題を全体の 1/3 以上**: 「エントリーしない判断」を覚えるのが初心者には最も重要。
4. **時間軸の明示**: チャート画像のタイムフレーム（1分足/5分足/日足）を必ず prompt に書く。
5. **板・出来高は補助情報**: チャートだけで判断できる問題から始め、徐々に板読みを要求する問題に進む。

## 選択肢の標準フォーマット

```json
{
  "choices": [
    { "id": "long", "label": "買い" },
    { "id": "short", "label": "空売り" },
    { "id": "no_trade", "label": "見送り" }
  ],
  "correct_choice_id": "no_trade"
}
```

3 択を基本とし、Phase 6 以降で「指値 vs 成行」「ロット数選択」など 4-5 択を導入。

## コピーライティングの語調

- ユーザへの呼びかけは「あなた」（カジュアル）
- 正解時: 「いいね！」「その通り！」「お見事！」
- 不正解時: 「惜しい！」「ここがポイント」（追い詰めない）
- 解説の冒頭は **必ず正解の選択肢を再掲** してから理由を述べる
