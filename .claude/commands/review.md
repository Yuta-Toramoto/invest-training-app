---
description: 直近の git diff をレビューし、リスク・規約違反・テスト漏れを指摘する。
---

以下を実行してください:

1. `git diff HEAD` でローカルの変更を取得
2. もしステージング済みのみレビューしたいなら `git diff --staged`
3. CLAUDE.md の鉄則と照らし合わせて、規約違反がないか確認:
   - ビジネスロジックが UI に漏れていないか
   - `packages/core` に React/Next の import がないか
   - `any` の使用
   - 1 ファイル 300 行超え
   - tRPC を経由しない DB アクセス
4. 以下を箇条書きで報告:
   - **🔴 Critical**: 規約違反、バグ、セキュリティリスク
   - **🟡 Warning**: 改善推奨、テスト漏れ
   - **🟢 Nice to have**: スタイル・命名の提案
5. テストが追加・更新されているか確認。なければ警告
6. 動作確認手順を提案

修正は勝手にしないこと。指摘のみ。
